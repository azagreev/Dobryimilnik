<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Alembic process -> PostgreSQL | Migration code mutates database schemas. |
| Settings -> database connection | Runtime config controls which database receives migrations. |
| CI runner -> database service | Automated checks apply migrations in disposable infrastructure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | Alembic migration scripts | mitigate | Keep baseline migration limited to approved schemas and make downgrade reversible. |
| T-01-06 | Information Disclosure | Database URL settings | mitigate | Load database URL from settings/environment; do not commit credentials in Alembic config. |
| T-01-07 | Denial of Service | Downgrade command | mitigate | Downgrade only Phase 1 schemas and never drop `public`. |
| T-01-08 | Repudiation | Migration history | mitigate | Use Alembic revision IDs and version table so applied migrations are auditable. |
</threat_model>
