<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner -> repository code | CI executes checked-in scripts and Makefile targets. |
| CI runner -> PostgreSQL service | Migration checks mutate disposable CI database. |
| Docker build -> backend runtime image | Build process packages backend code into an executable image. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-13 | Tampering | CI workflow commands | mitigate | CI invokes explicit Makefile targets and backend Docker build command. |
| T-01-14 | Information Disclosure | CI logs | mitigate | Do not print secret values; use test/local database credentials only in CI service environment. |
| T-01-15 | Denial of Service | CI migration checks | mitigate | Use disposable PostgreSQL service and downgrade after upgrade. |
| T-01-16 | Elevation of Privilege | Docker image build | mitigate | Build only backend image in Phase 1 and do not run privileged containers in CI. |
</threat_model>
