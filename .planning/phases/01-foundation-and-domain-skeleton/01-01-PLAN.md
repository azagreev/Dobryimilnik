<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host developer shell -> Makefile | Local commands can run containers and scripts. |
| Browser -> frontend service | Public local HTTP requests enter the Next.js dev service. |
| Browser/tooling -> backend API | Local HTTP requests enter FastAPI under `/api/v1`. |
| App services -> infrastructure containers | Backend and frontend depend on internal Compose networking. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Compose service discovery | mitigate | Use fixed internal service names and documented local ports; no business auth behavior in this phase. |
| T-01-02 | Tampering | Makefile command surface | mitigate | Keep commands explicit and repo-local; avoid shelling to unbounded scripts that are not committed. |
| T-01-03 | Information Disclosure | `.env` and local secrets | mitigate | Ignore `.env`, `.env.*`, and `secrets/local/`; do not create `.env.example`. |
| T-01-04 | Denial of Service | Local containers | accept | Local development containers can exhaust host resources; document `make down` recovery. |
</threat_model>
