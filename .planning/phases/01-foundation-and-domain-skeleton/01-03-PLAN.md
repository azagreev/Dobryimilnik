<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment -> backend settings | Runtime environment controls app behavior and connection URLs. |
| Secret files -> backend settings | Local files provide sensitive values. |
| Git repository -> deployment/runtime | Committed files must not contain secrets. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Information Disclosure | Secret files | mitigate | Ignore `secrets/local/` and read secrets through file paths, not committed values. |
| T-01-10 | Tampering | Environment selection | mitigate | Restrict `app_env` to `local` and `prod`; reject `staging` in Phase 1. |
| T-01-11 | Information Disclosure | Documentation | mitigate | Document secret filenames and creation process without sample secret values. |
| T-01-12 | Spoofing | External secret provider | accept | Yandex Lockbox is explicitly deferred; no production secret provider identity is trusted in this phase. |
</threat_model>
